Git提交使用GPG加密
2018-12-02

因为最近有空就会参与Codeignite4的开发工作，有时也会贡献自己的PR，CodeIgniter4的贡献指南明确规定
所有提交必须使用GPG进行签名，集成时，将踩过的坑分享之。

GPG是什么？Git提交为什么要使用GPG进行加密？

GnuPG(GNU Privacy Guard) 的简称，前世为PGP(Pretty Good Privacy)，是一款加密程序，主要用于
文本、邮件、目录、磁盘分区的加密，起源是为了防止信息泄露及被恶意监控。
GPG官网 GPG能做的非常多，更多信息可以Google进行了解。

Git 牵手 GPG，一对好基友从此隐匿网络

刚刚说过，CodeIgniter4的贡献指南中已经明确了所有的提交必须是经过GPG签名的，所以官方已经有了非常
详细的集成文档。传送门

  • 根据不同系统安装GPG软件包，并生成GPG Key。Mac OS 一行命令即可完成安装： brew install gnupg
  • 将生成的 GPG Key 添加到Github账户中
  • 提交时，使用 -S 参数对提交内容进行加密
    上面的文档已经写的非常详细了，根据文档操作都不会有问题。

我遇到的问题

在通过加密方式提交信息的时候，提示 error: gpg failed to sign the data，而且这个问题是第二次
遇到了，所以需要记录一下了。
遇到这个问题，首先要确认是否按文档把3个步骤都已经全部设置好了，那么仍然只需要一行命令即可解决：
export GPG_TTY=$(tty) 这样在当前的命令窗口内多次提交都可以用了，如果想一次性解决，只需要
把上面的命令添加在 ~/.bash_profile 文件中即可。
