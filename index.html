Codeigniter 4 环境搭建 2018-11-18|PHP

Codeigniter 4 介绍

目前最新的版本为alpha.2，基础功能的实现度已经很高了，目前每天的PR数量都在10个左右，但基本都是已代码优化升级、Bug修复为主。
我觉得开发人员可以优先体验了，这样有3个好处：

  • 提前熟悉 CI4（Codeigniter 4）的变化，从而快速适应，为以后的项目开发积累经验。
  • 目前 CI4 的文档、代码还没有特别成熟和稳定，在学习过程中，我们可以通过发现bug、修改bug来更深入的了解 CI4。
  • CI4 延续了 CI3 的特性，并没有像 Laravel 那样使用了非常多的设计模式、魔术方法等，依旧是短小精悍为主的小框架。这样我们学习曲线并没有那么陡。

安装前的环境准备

确认PHP版本大于 7.1 或更高

  1. 命令行模式通过php -version 确认PHP版本。(本机的 PHP 版本 7.2.9)

  2. 通过 phpinfo(); 方法确认PHP版本。
    在任意可访问的PHP文件中编写 phpinfo(); 页面会显示 PHP 版本详细信息。

确认是否安装 composer 依赖管理工具

  1. composer 是什么
    composer 是 PHP 世界里用于管理项目依赖的工具。相当于 Java 中的 Maven ，Python 中的 pip 。总之一句话，通过 Composer 可以方便、快捷的管理 PHP 项目中的各种包依赖关系，也可以非常方便的使用第三方开发包。必学！

  2. 如何确认 composer 信息
    在命令行模式输入 composer 按回车键，如出现相关版本信息及命令列表，既表示已经成功安装了

  3. 安装
    如没有显示以上信息，则需要安装 composer ，安装也非常简单，如果是Mac OS 只需要一行代码，其他系统需要在 composer官网 根据指引进行安装。
    当然现在有很多中文资料，比如 phpcomposer 包含了中文文档及全量的镜像。

安装 CI4 框架，成功运行后显示欢迎页面

  1. 使用 composer 命令行方式安装框架，进入指定的目录

  2. 在 Github 直接下载最新的 ZIP 文件直接使用
    点击下载

  3. 只用 Apache 或 Nginx 服务器运行，将运行根目录指向 CodeIgniter4/public 目录。
  4. 修改 application/Config/App.php 中 baseURL 变量值，设置为本地实际访问路径，例如 http://localhost
  5. 安装成功啦

如果你是Mac OS，可以使用 Valet 进行方便快捷的管理，下次分享。
